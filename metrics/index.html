Metrics

Read up on Metrics here.

Built-in metrics

MetricDescriptionRequires ground truth
exact_matchPrediction equals expected value (normalized string comparison).Yes
f1Token-level F1 (useful for extractive QA).Yes
bleuBLEU score vs. reference(s).Yes
latency_p9595th percentile latency in seconds (from runner metadata).No
customUser-defined Python function registered by name.Configurable

Using metrics in an eval

In your eval config, list the metric names. The pipeline will compute each and include them in the report.

metrics:
  - exact_match
  - f1
  - latency_p95

Custom metrics

Implement a function that takes (prediction, ground_truth?, example?) and returns a number or dict of numbers. Register it in the metrics registry.

def my_metric(prediction: str, ground_truth: str | None, example: dict) -> float:
    # Your logic here
    return score
Warning
Custom metrics should be deterministic. Avoid external API calls or randomness unless documented.
