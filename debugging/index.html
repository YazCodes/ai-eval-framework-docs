Debugging

When an eval fails or produces unexpected results, use this guide to narrow down the cause and fix it.

Common issues

SymptomLikely causeWhat to do
Pipeline fails on loadInvalid JSONL or schema mismatchValidate each line with jq or the dataset validator; check for trailing commas, encoding.
Runner timeouts / rate limitsAPI limits or slow modelAdd --delay, reduce --limit, or use a smaller model for dev.
Metrics always 0 or NaNWrong field names or missing ground truthConfirm expected (or equivalent) is present and metric reads the right keys.
Non-deterministic resultsModel sampling or no seedSet --seed and use temperature=0 (or equivalent) in runner config.

Debug checklist

  • ○Run with --limit 1 and inspect the single prediction and metric
  • ○Confirm dataset fields match what the runner and metrics expect
  • ○Check runner config (model name, API key env, endpoint)
  • ○Enable verbose/debug logging if available (e.g. LOG_LEVEL=debug)
  • ○Compare against a known-good run (same seed, same dataset)

Verbose output

LOG_LEVEL=debug python -m evals.run --eval my_eval --dataset data.jsonl --limit 3

Logs will print each example id, the prompt sent to the runner, and the raw prediction before metrics.

Tip
Add a small debug.jsonl (3–5 examples) next to your eval and use it for quick iteration without touching the full dataset.
